We are announcing a new Crystal release 1.19.2 with two regressions fixed.

See [the release notes of 1.19.0](/2026/01/15/1.19.0-released/) for all the
changes introduced in Crystal 1.19.

Pre-built packages are available on [GitHub
Releases](https://github.com/crystal-lang/crystal/releases/tag/1.19.2) and our
official distribution channels. See
[crystal-lang.org/install](https://crystal-lang.org/install/) for installation
instructions.

## Stats

This release includes [3 changes since
1.19.1](https://github.com/crystal-lang/crystal/pulls?q=is%3Apr+milestone%3A1.19.2)
by 2 contributors.  We thank all the contributors for all the effort put into
improving the language! ❤️

## Changes

### Security

`HTTP::Server` accepted requests containing both `Content-Length` and
`Transfer-Encoding` headers and prioritized `Content-Length`. This allowed HTTP
request smuggling as per [CWE-444] when the server is behind a vulnerable
frontend. Refer to the
[advisory](https://github.com/crystal-lang/crystal/security/advisories/GHSA-wqh5-7w63-pm68)
for more details.

`HTTP::Server` now rejects requests where both headers are present. The HTTP
parser now ignores `Content-Length` when the `Transfer-Encoding` header is
present (commit [c948b31]).

[CWE-444]: https://cwe.mitre.org/data/definitions/444.html
[c948b31]: https://github.com/crystal-lang/crystal/commit/c948b31ee6d5414050e771f89955c7dc02883ebc

### Regressions

This patch release fixes a regression in `Range#sample` ([#16866]) that could
eventually lose randomness.

For more details, visit the [full
changelog](https://github.com/crystal-lang/crystal/releases/tag/1.19.2).

[#16866]: https://github.com/crystal-lang/crystal/pull/16866
